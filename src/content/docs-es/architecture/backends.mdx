## Backend R1CS

### Estrategia de CompilaciĂłn

El backend mapea cada `SsaVar` a una `LinearCombination` (sparse `Vec<(Variable, FieldElement)>`).

**Operaciones gratis** (solo aritmĂŠtica de CL, sin restricciones):
- `Add(a, b)` â `lc_a + lc_b`
- `Sub(a, b)` â `lc_a - lc_b`
- `Neg(a)` â `-lc_a`
- `Const(v)` â `v * Variable::ONE`

**Operaciones que emiten restricciones:**
- `Mul(a, b)` â asignar cable, forzar `a Ă b = result` (1 restricciĂłn)
- `Div(a, b)` â calcular inverso, forzar `b Ă inv = 1`, luego `a Ă inv = result` (2 restricciones)
- `Mux(c, t, f)` â forzar `c` booleano, forzar `c Ă (t-f) = result - f` (2 restricciones)
- `AssertEq(a, b)` â forzar `a = b` (1 restricciĂłn)

|-----------|-------------|-------|

### OptimizaciĂłn Booleana

Si una variable estĂĄ en el conjunto `proven_boolean` (del pase `bool_prop`), el backend omite su restricciĂłn de verificaciĂłn booleana. Esto ahorra 1 restricciĂłn por variable booleana conocida usada en `Mux`, `And`, `Or` o `Not`.

## Backend Plonkish

**Archivo:** `zkc/src/plonkish_backend.rs`

El backend Plonkish compila IR en una tabla de filas de puerta con columnas, restricciones de copia y tablas de lookup.

### EvaluaciĂłn Perezosa

El backend Plonkish usa `PlonkVal` para evaluaciĂłn perezosa:


Las operaciones Add, Sub y Neg construyen expresiones diferidas. Solo cuando un valor se necesita para multiplicaciĂłn o una funciĂłn integrada, `materialize_val()` emite una fila real.

### Verificaciones de Rango vĂ­a Lookups

A diferencia de R1CS (que usa n+1 restricciones de descomposiciĂłn de bits), las verificaciones de rango Plonkish usan una **tabla de lookup**: 1 fila por verificaciĂłn independientemente del ancho de bits. La tabla de rango se pre-llena con valores `[0, 2^bits)`.

### Operaciones de ComparaciĂłn

- `IsLt(a, b)` â verificaciones de rango de 252 bits en ambos operandos + descomposiciĂłn de 253 bits de `b â a + 2^252 â 1`
- `IsLe(a, b)` â `1 â IsLt(b, a)` (intercambiar y negar)
- `IsZero(a)` â establece `d` a constante 0 (no es un ArithRow) para verificaciĂłn sĂłlida

### GeneraciĂłn de Testigos

`PlonkishWitnessGenerator` reproduce entradas `PlonkWitnessOp` para llenar la tabla de asignaciĂłn:

