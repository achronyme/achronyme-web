The Achronyme workspace contains 12 crates. All share the same version, edition 2021.

## Crate Overview

| Crate | Path | Purpose |
|-------|------|---------|
| `diagnostics` | `diagnostics/` | Shared diagnostic infrastructure: spans, errors, rendering (zero external deps) |
| `memory` | `memory/` | `FieldElement<F>` (BN254, BLS12-381, Goldilocks), `FieldBackend` trait, heap arenas, GC |
| `achronyme-parser` | `achronyme-parser/` | Recursive descent parser, owned AST types |
| `constraints` | `constraints/` | R1CS constraint system, Plonkish system, Poseidon hash (multi-field), binary export |
| `ir` | `ir/` | SSA intermediate representation, AST→IR lowering, optimization passes, ProveIR |
| `akron` | `akron/` | Register-based general-purpose bytecode VM, interpreter, prototype method dispatch |
| `artik` | `artik/` | Dedicated witness-computation VM (SSA-style, no heap, no GC) |
| `compiler` | `compiler/` | R1CS and Plonkish backends, witness generation |
| `proving` | `proving/` | Groth16 (arkworks), PlonK (halo2-KZG), Solidity verifier generation |
| `achronyme-std` | `std/` | Standard library via `NativeModule` trait (parse_int, join, I/O) |
| `ach-macros` | `ach-macros/` | Proc-macros: `#[ach_native]`, `#[ach_module]` |
| `circom` | `circom/` | Circom 2.x frontend: lexer, parser, analysis, ProveIR lowering |
| `cli` | `cli/` | Command-line interface, proof orchestration |

## Dependency Graph

```
diagnostics (zero deps)
  │
  ├──→ achronyme-parser
  │         │
  ├──→ ir ──┘──→ memory, constraints, artik
  │    │
  ├──→ circom ──→ ir, artik
  │
  ├──→ akron → memory, constraints
  │
  └──→ compiler → ir, achronyme-parser, memory, constraints, artik
          │
          └──→ cli → compiler, akron, proving
```

### Foundation (no workspace dependencies)

**`diagnostics`** — Shared error reporting infrastructure with zero external dependencies. Defines `Span` (source location), `Diagnostic` (unified error/warning type with builder pattern), `SpanRange`, `Severity`, `Label`, `Suggestion`, and `DiagnosticRenderer` (rustc-style source rendering with ANSI colors). Consumed by all frontends and the IR crate.

**`memory`** — The base layer. Defines `FieldElement<F: FieldBackend>`, a generic field element parameterized by a zero-sized backend marker type. Three backends are implemented: `Bn254Fr` (Montgomery `[u64; 4]`), `Bls12_381Fr` (Montgomery `[u64; 4]`), and `GoldilocksFr` (direct `u64`). The default type parameter is `Bn254Fr`, so bare `FieldElement` means BN254. Also provides tagged `Value` (4-bit tag + 60-bit payload), typed heap arenas, and mark-and-sweep GC.

**`achronyme-parser`** — Achronyme language parser. Recursive descent with Pratt expression parsing. Re-exports all types from `diagnostics`. Exports `parse_program(source) -> (Program, Vec<Diagnostic>)` and owned AST types.

### Middle Layer

**`constraints`** → `memory`

Core constraint system abstractions. `Variable(usize)` for wire references, `LinearCombination` for sparse symbolic expressions, `ConstraintSystem` for A×B=C enforcement. Also contains the `PlonkishSystem` (gates, lookups, copy constraints), Poseidon hash parameters (circomlibjs-compatible), and binary export (`write_r1cs`, `write_wtns` in iden3 format).

**`ir`** → `memory`, `constraints`, `diagnostics`, `achronyme-parser`

SSA intermediate representation for circuit compilation. `IrLowering` converts the AST into a flat `IrProgram` (19 instruction variants). Includes 4 optimization passes (`const_fold`, `dce`, `bool_prop`, `taint`) and an evaluator for concrete IR execution.

**`circom`** → `diagnostics`, `ir`

Circom 2.x frontend. Complete pipeline from `.circom` source to ProveIR:
- **Lexer** — single-pass tokenizer with maximal munch disambiguation for signal operators (`<==`, `<--`, `===`, `==>`, `-->`)
- **Parser** — hand-written Pratt + recursive descent, matching Circom 2.x syntax (templates, signals, components, buses)
- **Analysis** — constraint pairing check (`<--` without `===` = hard error E100), include resolution with cycle detection
- **Lowering** — signal extraction, expression/statement/template lowering to ProveIR `CircuitNode`/`CircuitExpr`

**`akron`** → `memory`, `constraints`

General-purpose register-based bytecode VM. 40+ opcodes, bytecode compiler (`FunctionCompiler`), 40+ native functions, `prove {}` block support. Uses `constraints` for Poseidon hash computation in VM mode. See [Akron VM & Bytecode](./akron-vm).

**`artik`** → `memory`

Dedicated witness-computation VM paired with Akron. SSA-style register file, ~25 opcodes, no heap, no GC. Lifted from Circom function bodies that cannot be circuit-inlined (closes the E212 gap). See [Artik Witness VM](./artik-vm).

### Top Layer

**`compiler`** → `memory`, `akron`, `constraints`, `achronyme-parser`, `ir`, `artik`

Backend compilation from IR to constraint systems. `R1CSCompiler` maps SSA variables to linear combinations and emits R1CS constraints. `PlonkishCompiler` uses lazy `PlonkVal` evaluation with cell-based rows. Both backends include `compile_ir_with_witness()` for combined compilation + witness generation.

**`cli`** → `memory`, `akron`, `compiler`, `constraints`, `ir`

User-facing CLI. Dispatches `ach run` (VM path) and `ach circuit` (constraint path). Handles proof orchestration with external cryptographic libraries:
- **arkworks**: `ark-groth16`, `ark-bn254`, `ark-relations` for Groth16 proving
- **halo2**: `halo2_proofs` for PlonK proving

## Key Entry Points

| Task | Crate | Function |
|------|-------|----------|
| Parse source | `achronyme-parser` | `parse_program(source)` |
| Lower to IR | `ir` | `IrLowering::lower_circuit(source, pub, wit)` |
| Optimize IR | `ir` | `passes::optimize(&mut program)` |
| Compile to R1CS | `compiler` | `R1CSCompiler::compile_ir(program)` |
| Compile to Plonkish | `compiler` | `PlonkishCompiler::compile_ir(program)` |
| Generate witness | `compiler` | `R1CSCompiler::compile_ir_with_witness(program, inputs)` |
| Export .r1cs | `constraints` | `write_r1cs(cs)` |
| Export .wtns | `constraints` | `write_wtns(witness)` |
| Run VM | `akron` | `VM::interpret(function)` |
| Execute witness VM | `artik` | `artik::execute(program, ctx)` |
| Run CLI | `cli` | `main()` → command dispatch |
