## Overview

The `circom` crate is the Achronyme implementation of the Circom 2.x compiler ‚Äî frontend only. Circuit semantics are lowered to ProveIR, then handed off to the same instantiation ‚Üí IR ‚Üí backend pipeline as `prove {}` blocks.

For user-facing usage (importing templates, calling them from `.ach`), see the [Circom Interop](/docs/circom/overview/) chapter. This page documents the **internals** for contributors.

## Pipeline

```
.circom source

    ‚îÇ
    ‚ñº  circom::lowering

    ‚îÇ
    ‚ñº  ProveIR instantiate

    ‚îÇ
    ‚ñº  R1CS backend

```

Entry point: `compile_to_prove_ir(source: &str) -> Result<CircomCompileResult, CircomError>` (file: `crates/circom/src/lib.rs`).

```rust
pub struct CircomCompileResult {
    pub prove_ir: ProveIR,
    pub output_names: HashSet<String>,
    pub capture_values: HashMap<String, u64>,
    pub warnings: Vec<Diagnostic>,
}
```

Multi-file API: `compile_file(path, library_dirs)` resolves `include` chains with mutual deduplication (`-l/--lib` CLI flag, `[circom] libs` in `achronyme.toml`).

## Lexer

File: `crates/circom/src/lexer.rs`. Single-pass byte scanner. Maximal-munch disambiguation for the signal-operator family:

| Tokens | Meaning |
|--------|---------|
| `<` | less-than |
| `<=` | less-or-equal |
| `<==` | constrained signal assignment (LHS computed from RHS, also constrained) |
| `<--` | unconstrained signal assignment (witness hint, no constraint emitted) |
| `===` | bare constraint |
| `==>` | constrained signal assignment, reverse direction |
| `-->` | unconstrained signal assignment, reverse direction |

The lexer must commit to the longest match; otherwise `<==` would be lexed as `<=` followed by `=`.

## Parser

File: `crates/circom/src/parser/`. Hand-written Pratt + recursive descent for Circom 2.x. Produces:

```rust
pub struct CircomProgram {
    pub version: Option<PragmaVersion>,
    pub definitions: Vec<Definition>,
}

pub enum Definition {
    Template { name, params, body: TemplateBody },
    Function { name, params, body: FunctionBody },         // imperative, compile-time evaluator
    Bus { ‚Ä¶ },
    Component { name, component_type, params: Vec<Expr> }, // top-level main
    Include { path },
}

pub struct TemplateBody {
    pub signals: Vec<SignalDecl>,
    pub statements: Vec<Stmt>,
}

pub enum SignalDecl {
    Input { name, array_size: Option<Expr> },
    Output { name, array_size: Option<Expr> },
    Intermediate { name, array_size: Option<Expr> },
}
```

Bare-statement form is supported (Circom 2.1+): `for (...) stmt;` without braces.

## Analysis (`circom::analysis`)

File: `crates/circom/src/analysis/`. Runs after parsing:

1. **Constraint pairing check** ‚Äî every `<--` / `-->` (unconstrained assignment) must have a corresponding `===` constraint over the same signal. Bare `<--` without `===` is the #1 source of under-constrained vulnerabilities ‚Äî emitted as warning **W103** (downgraded from hard error E101 in beta.20 to allow if-else branching patterns where one branch constrains).
2. **Include resolution** ‚Äî DAG-walks `include "file.circom";` chains, detects cycles, deduplicates mutual includes.
3. **Main component validation** ‚Äî exactly one `component main { ‚Ä¶ }` declaration is required (E210 if missing, E211 if it references an undefined template).

## Lowering (`circom::lowering`)

File: `crates/circom/src/lowering/`. Translates the validated Circom AST into ProveIR.

Submodules (post-audit split, 21 modules):

- `lowering/mod.rs` ‚Äî driver
- `lowering/components.rs` ‚Äî component instantiation, parameter binding
- `lowering/signals.rs` ‚Äî signal dimension extraction
- `lowering/expressions.rs` ‚Äî `CircuitExpr` lowering
- `lowering/statements.rs` ‚Äî `CircuitNode` lowering
- `lowering/const_fold.rs` ‚Äî compile-time constant evaluation (BigVal, ternary fold, array constants)
- `lowering/artik_lift.rs` ‚Äî function bodies that can't inline ‚Üí Artik bytecode
- `lowering/context.rs` ‚Äî environment, captures, known_constants

Behaviors that matter:

- **`function` keyword** ‚Äî compile-time interpreter for imperative bodies (`var`, `while`, `for`, `if/else`, `return`, `++`, `*=`, nested calls). Unlocks `nbits()`, `log2()`, etc. Result fed to `precompute_vars` to resolve signal dimensions.
- **Ternary constant-fold** ‚Äî compile-time-known ternaries select a branch at lowering, avoiding dead-branch `ArrayIndex` errors like `xL[-1]`.
- **For-loop unrolling** ‚Äî auto-unrolls when the body references `known_array_values`. ForRange variants: `Literal`, `WithCapture` (param-bound), `WithExpr` (computed bound, e.g., `n+1`).
- **Component arrays** ‚Äî `component muls[n]; muls[i] = Template()` unrolls at lowering using `known_constants`.
- **Array template params** ‚Äî `Ark(t, C, 0)` passes arrays through component inlining; `Capture` resolves from parent params.
- **2D arrays** ‚Äî `var M[t][t] = GET_MATRIX(t)` ‚Äî flattened with strides + multi-dim index resolution.
- **BigVal evaluator** ‚Äî compile-time `var` is two's-complement 256-bit `[u64; 4]`. Fixes `1 << 128` overflow in `CompConstant`.

## Component composition

A component instantiation:

```circom
component lt = LessThan(8);
lt.in[0] <== a;
lt.in[1] <== b;
out <== lt.out;
```

‚Ä¶ expands at lowering time: lt's body is inlined with `(8)` substituted for the template parameter `n`, lt's input signals become let-bindings on the parent's expression tree, lt's output signal feeds back to `out` via `===`.

Captures from the parent flow into the child via `CaptureDef`. Multi-level nesting (e.g., `EscalarMulAny(254)` containing `Pedersen` and `EscalarMulFix`) was the focus of the deep-inlining bug fixes (2026-04-04).

## Witness hints

`<--` (and `-->`) emit `WitnessHint { name, hint }` nodes in ProveIR. They give the prover a value but emit no constraint. Combined with `===`, they implement the standard "witness + check" pattern.

For loops with array signal assignment (`out[i] <-- expr`), the lowering emits `WitnessHintIndexed`.

## Diagnostic codes

Circom-specific ranges:

| Range | Severity | Meaning |
|-------|----------|---------|
| E100‚ÄìE102 | Error | Parser errors (syntax, malformed signal ops) |
| W101‚ÄìW103 | Warning | Version pragma issues, deprecated forms, double-assignment (W103) |
| E200‚ÄìE211 | Error | Lowering errors (undefined template, missing main, type mismatch, dim mismatch) |
| E300‚ÄìE306 | Error | Constraint analysis errors (under-constrained signals, structural issues) |

Selected codes:

- **E101** ‚Äî historical; downgraded to W103 in beta.20.
- **E210** ‚Äî `NoMainComponent`: no `component main` declaration found.
- **E211** ‚Äî `MainTemplateNotFound`: `main` references an undefined template.
- **W103** ‚Äî `DoubleSignalAssignment`: a signal is assigned twice (allowed for if-else branch coverage).

All diagnostics carry "did you mean?" Levenshtein suggestions and use the standard `DiagnosticRenderer` for rustc-style output.

## Compile-time constant evaluation

- Function bodies are evaluated at compile time when their callers need a constant (e.g., signal dimensions).
- `EvalValue` ‚àà `{Scalar(BigVal), Array(Vec<EvalValue>), Expr(CircuitExpr)}`.
- `eval_function_to_value` runs the imperative interpreter; if-else branches return distinct values are unified.
- `precompute_array_vars` expands array `var C[n] = POSEIDON_C(t)` into per-element `Const` lets.

## Artik lift

For function bodies that **can't** be inlined as `CircuitExpr` (because they have signal-dependent control flow or array writes), the lowering hands them to `circom::lowering::artik_lift`. Output: an Artik bytecode blob, embedded in `CircuitNode::WitnessCall`.

See [Artik Witness VM](/docs/architecture/artik-vm/) for the dispatch path.

## R1CS optimizer integration

The Circom frontend produces ProveIR that, after instantiation, is optimized by **O1 + O2 passes**:

| Pass | Effect |
|------|--------|
| Constant propagation | BN254 field folding + const-signal inlining |
| Linear elimination | Gaussian-style substitution |
| Zero-product handling | Removes redundant `x ¬∑ y == 0` when one side is provably zero |
| DEDUCE | Frequency heuristic + tautological linear removal |

Result: Achronyme matches or beats `circom --O2` on every reference template (Num2Bits 9 vs 17, LessThan 10 vs 20, MiMCSponge 1317 vs 1320, EscalarMulAny 2310 = circom).

## Adversarial soundness tests

File: `crates/circom/tests/adversarial.rs`. Four tests prove constraint deltas vs circom O2 are LC folding wins, not under-constrained bugs:

1. Forge non-bool bits in `Num2Bits` ‚Üí assertion fails.
2. Forge sum violation in `Num2Bits` ‚Üí assertion fails.
3. Forge `LessThan` output (both directions) ‚Üí assertion fails.
